Webサイト修正・LP/小規模サイト制作・Excelマクロ（VBA）解読/修正を、**プラットフォームを介さず直接お受けする**ための窓口サイト。`index.html` 1ファイル完結（フレームワーク不使用・静的）。

公開先（予定）：`https://hayaboosan.github.io/<リポジトリ名>/`
作品集（ポートフォリオ）：`https://hayaboosan.github.io/portfolio/` ← **これとは別サイト**

## ⚠️ 運用ルール（最重要・必ず守る）

このサイトは「作品集」とは**完全に分離**して運用する。理由は、クラウドソーシング各社が「自社で出会った客を外部に誘導して直接取引すること」を規約で禁止しているため（CW＝違約金100万円 or 利用料相当 / ランサーズ＝リピート取引も禁止 / ココナラ＝予告なしアカウント停止）。

やってよいこと / ダメなこと：

- ❌ このURLを **クラウドワークス・ランサーズ・ココナラのプロフィール／作品URL／メッセージに載せない**。
- ❌ **ポートフォリオ（portfolio）からこのサイトへリンクを張らない**。ポートフォリオはプラットフォームに登録済みなので、そこに直接受注の導線があると「外部誘導」とみなされる。
- ❌ プラットフォームで出会った発注者を、このサイトへ誘導しない。
- ✅ このURLは **紹介・知人・SNS・名刺など、プラットフォーム外で新規に出会った相手にだけ** お渡しする。
- ✅ この窓口サイト → ポートフォリオ へのリンクはOK（自分のサイト内で実績を見せるだけ。逆向きはNG）。
- ✅ 検索除けに `<meta name="robots" content="noindex, nofollow">` を入れてある。URLを知っている人だけが到達する。**この行は消さない。**

> まとめ：「プラットフォーム客の引き抜き」＝規約違反。「別ルートの新規客を自分のサイトで受ける」＝OK。この線を越えないこと。

## セットアップ（フォームの送信先・約2分）

フォームは [Web3Forms](https://web3forms.com/)（**アカウント登録不要・無料・無制限**）でメール送信する。送信先メールは公開ソースに出ず、ランダムなキー（UUID）だけが載るので、メールアドレスは隠れる。

1. <https://web3forms.com/> を開き、**受信したいメールアドレス**（例：Gmail）を入力して「Create Access Key」。
2. 送られてくる Access Key（`xxxxxxxx-xxxx-...` の形）をコピー。
3. `index.html` の次の1行のプレースホルダを、そのキーに置き換える：
   ```html
   <input type="hidden" name="access_key" value="YOUR_WEB3FORMS_ACCESS_KEY" />
   ```
4. 公開後、自分でテスト送信して、メールが届くことを確認する。

> キー未設定のままだと、送信ボタンを押しても「送信先が未設定です」と表示され、誤って空送信されない作りにしてある。

### 送信先を別サービスにしたい場合（任意）

- **Formspree**：`<form action="https://formspree.io/f/＜フォームID＞">` に変更（要無料登録）。`access_key` の hidden は削除。
- **Tally / Googleフォーム**：このフォーム部分を各サービスの埋め込み iframe に差し替えてもよい（送信処理のJSは不要になる）。

いずれも「サイト上で決済まで完結させない（＝見積もり依頼を受けるだけ）」点は変えないこと（下の法務メモ参照）。

## 公開（GitHub Pages）

ポートフォリオと同じ要領（`hayaboosan` アカウント・`gh` CLI）：

```sh
cd C:\Users\Hayabusa\Works\contact-site
git init
git add -A
git commit -m "Add direct order / quote-request site"
gh repo create <リポジトリ名> --public --source=. --remote=origin --push
# GitHub Pages を有効化（main / root）
gh api -X POST repos/hayaboosan/<リポジトリ名>/pages -f "source[branch]=main" -f "source[path]=/" 2>$null
```

- リポジトリ名がそのままURLのパスになる（例：`desk` → `hayaboosan.github.io/desk/`）。あまり推測されにくい名前にしてもよい。
- 静的サイトなのでビルド不要。push の数十秒後に反映。

## 支払い・法務メモ（調査済みの要点）

- **支払い**：銀行振込。**着手金（前払い）＋納品後の残金**の分割を基本に。本人名義の個人口座でOK（屋号口座は不要）。
- **取引条件の明示**：業務委託は「フリーランス・事業者間取引適正化等法」で、報酬額・支払期日・支払方法などを**書面（メール可）で明示する義務**がある。簡単な発注内容メールでよいので、着手前に必ず渡す。
- **特定商取引法の表記**：このサイトは**「見積もり依頼を受けるだけ（サイト上で申込み・決済を完結させない）」**ので、原則「通信販売」に当たらず**特商法表記（氏名・住所・電話の掲示）は不要**＝本名・自宅住所を出さなくてよい。
  - ⚠️ 将来 Stripe / Square / BASE 等で**サイト上に決済・購入ボタンを置く**と「通信販売」に該当し、特商法表記が必要になる（住所・電話は「請求があれば開示」運用やバーチャルオフィスで省略可だが、本名は原則必要）。決済を入れるときは要再検討。
- **著作権・機密**：納品物の権利と利用範囲は契約時に取り決め。預かったデータは目的外利用しない（NDA対応）。社外秘ソースは扱いを事前合意。
